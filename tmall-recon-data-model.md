# 天猫对账数据材料关联模型（2026年2-4月）

## 核心改变（v2）

**匹配策略**：不再使用 (doc_no, norm_product) 商品级精确匹配，改为：
```
KSOA doc_no → outbound_fallback → sys_order → wln_sys_to_origin → [tmall_orders]
```
一个KSOA单据对应一个或多个天猫订单，在doc维度聚合后统一比对。

**等式验证**：
```
KSOA小计 = 支付宝到账合计 + 聚合结算到账合计
```
平台费用独立展示，不参与等式验证。

## 二、数据串联关系（v2）

```
KSOA零售-天猫.xls
  └─ 单据编号（doc_no）
       │
       └─ 万里牛销售出库退货明细.xlsx
            └─ 出入库单号 = doc_no → 取第一条记录的系统订单号
                 │
                 └─ 万里牛订单明细.xlsx
                      └─ 系统单号 → 原订单号（天猫订单号）
                           │
                           │  注意：一个sys_order可能有多行记录
                           │  每行的原订单号是一个天猫订单号
                           │  多天猫订单用 | 分隔或分多行
                           │
                           ├─ 天猫订单明细报表.xlsx → 买家实付、订单状态
                           ├─ 支付宝交易订单明细.xls → 商家实收
                           ├─ 支付宝资金流水.xlsx → 在线支付收款、分账费用
                           └─ 聚合结算账户收支明细.xlsx → 交易收款、扣款、退款
```

---

## 三、关键字段说明

### 支付宝资金流水
- **在线支付**（收入）：买家付款到支付宝
- **分账/转账**（支出）：平台收取的服务费/分账手续费
- **业务基础订单号**：用于匹配天猫订单

### 聚合结算账户收支明细
- **交易收款**（收入）：聚合账户收到的款项
- **扣款退回**（收入）：之前扣款被退回
- **扣款**（支出）：平台扣减
- **交易退款(售后)**（支出）：买家退货退款

### 万里牛订单明细 原订单号字段
- 格式可能是单个订单号：`3216781059386934156`
- 也可能是多个订单号用`|`分隔：`3214663032817861283|3214209867403861283`
- 也可能分多行记录同一个sys_order的不同tmall订单

## 五、等式验证规则

### 核心等式
```
KSOA小计 = 支付宝到账合计 + 聚合结算到账合计
```

- KSOA小计 = 同一doc下所有行实收金额之和
- 支付宝到账合计 = 关联所有tmall订单的在线支付收入之和
- 聚合结算到账合计 = 关联所有tmall订单的交易收款+扣款退回之和

### 净收（独立）
```
净收 = KSOA小计 + 平台费用支出
```
平台费用 = 支付宝分账/转账支出 + 聚合结算扣款/退款（负数）
