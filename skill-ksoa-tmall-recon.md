```

  └─ 单据编号 = 万里牛销售出库退货明细.出入库单号
       └─ 系统订单号（万里牛内部单）
            └─ 万里牛订单明细.系统单号 → 原订单号（天猫订单号，可能多个，用|分隔）
                 │
                 ├─ 天猫订单明细报表.订单编号 → 订单状态/买家实付/时间

                 ├─ 支付宝资金流水.业务基础订单号 → 在线支付收款+分账/转账费用
```

**关键**：万里牛订单明细中，一个系统单号可能对应多行，每行的`原订单号`是一个天猫订单号。如果一个系统单对应多个天猫订单（合并发货），会在万里牛订单明细中出现多条记录。

## 三、匹配策略（核心）

### 问题背景
KSOA是商品维度（一笔订单拆多条KSOA记录），天猫订单是订单维度。旧方案试图在商品行级别匹配tmall订单，导致维度混淆、匹配错误。

### 正确策略：按KSOA单据编号聚合，不在商品行级别匹配

**匹配链路（两步，无商品维度匹配）：**
```
KSOA单据编号
  └─ outbound_fallback[doc_no] → 万里牛系统订单号
       └─ wln_sys_to_origin[sys_order] → [天猫订单号列表]（去重）
```

**不再使用 (doc_no, norm_product) 精确匹配**，因为：
1. 万里牛订单明细已经记录了每个系统单号对应的所有天猫订单
2. 商品维度匹配容易因名称标准化不完美而匹配错误
3. 合并发货时一个doc对应多个tmall订单，在商品行级别拆分会丢失整体性

**wln_sys_to_origin 构建逻辑：**
```python
wln_sys_to_origin = defaultdict(set)  # 一个sys_order可能有多行
for row in 万里牛订单明细:
    sys_order = row['系统单号']
    origin = row['原订单号']  # 可能是 "3214663032817861283|3214209867403861283"
    if '|' in origin:
        for t in origin.split('|'):
            wln_sys_to_origin[sys_order].add(t.strip())
    else:
        wln_sys_to_origin[sys_order].add(origin)
# 转为 list，去重
wln_sys_to_origin = {k: list(v) for k, v in wln_sys_to_origin.items()}
```

## 四、数据验证公式

### 核心等式
```
KSOA小计 = 支付宝到账合计 + 聚合结算到账合计
```

其中：
- **KSOA小计**：同一KSOA单据编号下所有行实收金额之和
- **支付宝到账合计**：该doc关联的所有天猫订单的支付宝资金流水`在线支付`收入之和
- **聚合结算到账合计**：该doc关联的所有天猫订单的聚合结算`交易收款+扣款退回`之和

**净收计算（独立展示，不参与等式验证）：**
```
净收 = KSOA小计 + 平台费用支出
平台费用支出 = 支付宝资金流水非在线支付支出 + 聚合结算扣款/退款（负数）
```

**业务含义：**
- KSOA小计是商家实际收到的人民币金额（不含平台后续扣减）
- 平台费用是支付宝/聚合账户在到账后额外收取的分账/服务费
- 净收 = 到账金额 - 平台费用，是商家实际到手的金额

## 五、差异分类与处理

### 等式差异 = KSOA小计 - (支付宝到账 + 聚合结算到账)

| 差异类型 | 描述 | 处理方式 |
|---|---|---|
| **无差异** | \|等式差异\| ≤ 0.01 | 无需处理 |
| **含关闭订单** | 关联的天猫订单中有状态=交易关闭 | 橙色标注，参与等式验证，需核查关闭订单的KSOA为何有记录 |
| **正常订单差异** | 等式差异 > 0.01 且无关闭订单 | 红色标注，需逐单核查原因 |

### 交易关闭订单说明
订单关闭不代表没有平台费用支出。有些订单关闭后，平台仍可能收取分账费用。因此：
- 交易关闭订单**参与等式验证**，不排除
- 标注`含关闭订单`提醒核查
- 差异原因可能是：KSOA记录了销售但订单关闭无付款、或者KSOA无记录但平台已退款等

## 十、已知问题与处理方式

### Q1: 为什么有这么多含关闭订单的差异？
A: 交易关闭订单的买家实付为0，但KSOA记录了销售出库。可能是万里牛预发货/锁库存但订单最终未付款成功。需要核查这些KSOA记录是否应该确认收入。

### Q2: 为什么KSOA有金额但支付宝到账为0？
A: 两种情况：①订单关闭，无付款；②支付宝到账时间晚于对账截止日（正常差异，可后续跟踪）。

### Q3: 为什么支付宝到账金额和KSOA小计有零头差异（如12.61元）？
A: 可能是平台优惠/积分/红包等在天猫侧抵扣但未在KSOA侧体现，或四舍五入差异。需要逐单核查。

### Q4: 合并发货订单（一个KSOA对应多个天猫订单）如何处理？
A: 新方案直接按doc聚合，所有关联的天猫订单金额合计后与KSOA小计比对。不需要在商品行级别拆分归属。
